# 総合演習：図書・備品管理システム

本章で学習したJavaのオブジェクト指向機能（ラッパークラス、Generics、抽象クラス、列挙型）を組み合わせて、図書や備品を管理するシステムを設計・実装しましょう。
この演習を通じて、これまでの知識が実際のシステム設計においてどのように組み合わされて機能するかを理解します。

## 演習

### 演習1
貸出ステータスを管理する列挙型 `LendingStatus`、貸出物の共通の性質を定義する抽象クラス `LendingItem`、および具象クラス `Book` と `Device` を作成しなさい。
各クラスは以下の要件を満たすように設計しなさい。

1. 列挙型 `LendingStatus`
- 貸出可能な状態を表す `AVAILABLE` を定義しなさい。
- 貸出中の状態を表す `LENT` を定義しなさい。
- メンテナンス中の状態を表す `MAINTENANCE` を定義しなさい。

2. 抽象クラス `LendingItem`
- フィールドとして、IDを表す `id`（String型）、タイトルを表す `title`（String型）、貸出状態を表す `status`（`LendingStatus` 型）、および貸出中の利用者IDを表す `borrowerId`（`Integer` 型）を定義しなさい。
- `borrowerId` は、未貸出の場合は値が存在しない状態を表すために、プリミティブ型ではなくラッパークラスを使用しなさい。
- コンストラクタで `id` と `title` を受け取り、初期状態として `status` に `LendingStatus.AVAILABLE` を、`borrowerId` に `null` を代入しなさい。
- 外部から各フィールドを参照するためのゲッターメソッド（`getId()`、`getTitle()`、`getStatus()`、`getBorrowerId()`）および `status` と `borrowerId` を変更するためのセッターメソッド（`setStatus(LendingStatus status)`、`setBorrowerId(Integer borrowerId)`）を定義しなさい。
- 貸出物の最大貸出日数を返す抽象メソッド `getMaxLendingDays()` を定義しなさい。

3. 具象クラス `Book`（`LendingItem` を継承する）
- 独自のフィールドとして、著者を表す `author`（String型）を定義しなさい。
- コンストラクタで `id`、`title`、`author` を受け取り、親クラスのコンストラクタを呼び出すとともに、`author` を初期化しなさい。
- `getMaxLendingDays()` をオーバーライドし、書籍の最大貸出日数として `14` を返しなさい。
- 著者を隠蔽しつつ取得できるように、ゲッターメソッド `getAuthor()` を定義しなさい。

4. 具象クラス `Device`（`LendingItem` を継承する）
- 独自のフィールドとして、保管場所を表す `location`（String型）を定義しなさい。
- コンストラクタで `id`、`title`、`location` を受け取り、親クラスのコンストラクタを呼び出すとともに、`location` を初期化しなさい。
- `getMaxLendingDays()` をオーバーライドし、機器の最大貸出日数として `7` を返しなさい。
- 保管場所を隠蔽しつつ取得できるように、ゲッターメソッド `getLocation()` を定義しなさい。

### 演習2
演習1で作成した貸出物を管理する、Genericsを用いた管理クラス `LendingManager<T extends LendingItem>` を作成しなさい。
このクラスは以下の要件を満たすように設計しなさい。

1. 独自の型パラメータ `T` は、`LendingItem` またはそのサブクラスのみを受け付けるように制限しなさい。
2. 登録されたアイテムを保持するリストとして `ArrayList<T>` を定義しなさい。
3. 以下のメソッドを実装しなさい。
- アイテムを登録する `register(T item)` メソッド
- 指定されたIDのアイテムを貸し出す `lendItem(String id, int borrowerId)` メソッド。対象のアイテムが存在し、かつステータスが `AVAILABLE` である場合のみ、ステータスを `LENT` に変更し、`borrowerId` に引数の値を設定して `true` を返しなさい。それ以外の場合は `false` を返しなさい。
- 指定されたIDのアイテムを返却する `returnItem(String id)` メソッド。対象のアイテムが存在し、かつステータスが `LENT` である場合のみ、ステータスを `AVAILABLE` に変更し、`borrowerId` に `null` を設定して `true` を返しなさい。それ以外の場合は `false` を返しなさい。
- 登録されているすべてのアイテムのID、タイトル、ステータス、および貸出中の場合は借用者IDと最大貸出可能日数を表示する `showItems()` メソッド

### 演習3

## 演習の解答例

### 演習1の解答例

貸出ステータスを定義した列挙型 `LendingStatus` の実装例は以下の通りである。
状態を表す3つの定数を定義している。

```java:LendingStatus.java
enum LendingStatus {
  AVAILABLE,
  LENT,
  MAINTENANCE
}
```

貸出物の共通の属性と操作を定義した抽象クラス `LendingItem` の実装例は以下の通りである。
未貸出の状態を表現するために、利用者IDを保持するフィールドには `Integer` 型を使用し、初期値として `null` を代入している。
また、カプセル化の原則に従い、フィールドはすべて `private` で宣言し、外部からのアクセスにはゲッターとセッターを提供している。
クラス内部のメンバにアクセスする際は `this.` を明示的に記述している。

```java:LendingItem.java
abstract class LendingItem {
  private String id;
  private String title;
  private LendingStatus status;
  private Integer borrowerId;

  public LendingItem(String id, String title) {
    this.id = id;
    this.title = title;
    this.status = LendingStatus.AVAILABLE;
    this.borrowerId = null;
  }

  public String getId() {
    return this.id;
  }

  public String getTitle() {
    return this.title;
  }

  public LendingStatus getStatus() {
    return this.status;
  }

  public Integer getBorrowerId() {
    return this.borrowerId;
  }

  public void setStatus(LendingStatus status) {
    this.status = status;
  }

  public void setBorrowerId(Integer borrowerId) {
    this.borrowerId = borrowerId;
  }

  public abstract int getMaxLendingDays();
}
```

書籍を表現する具象クラス `Book` の実装例は以下の通りである。
コンストラクタでは `super` を用いて親クラスのコンストラクタを呼び出している。
`getMaxLendingDays` メソッドをオーバーライドし、書籍の貸出上限である `14` を返している。

```java:Book.java
class Book extends LendingItem {
  private String author;

  public Book(String id, String title, String author) {
    super(id, title);
    this.author = author;
  }

  public String getAuthor() {
    return this.author;
  }

  @Override
  public int getMaxLendingDays() {
    return 14;
  }
}
```

機器を表現する具象クラス `Device` の実装例は以下の通りである。
`getMaxLendingDays` メソッドをオーバーライドし、機器の貸出上限である `7` を返している。

```java:Device.java
class Device extends LendingItem {
  private String location;

  public Device(String id, String title, String location) {
    super(id, title);
    this.location = location;
  }

  public String getLocation() {
    return this.location;
  }

  @Override
  public int getMaxLendingDays() {
    return 7;
  }
}
```

### 演習2の解答例

Genericsを用いて設計した管理クラス `LendingManager` の実装例は以下の通りである。
型パラメータ `T` を `T extends LendingItem` と制限することで、異なる具象クラスに対しても同一の貸出管理ロジックを適用できるようにしている。
貸出および返却処理では、指定されたIDを持つアイテムを走査し、現在のステータスに応じて適切に状態を更新している。

```java:LendingManager.java
import java.util.ArrayList;

class LendingManager<T extends LendingItem> {
  private ArrayList<T> items;

  public LendingManager() {
    this.items = new ArrayList<T>();
  }

  public void register(T item) {
    this.items.add(item);
  }

  public boolean lendItem(String id, int borrowerId) {
    for (T item : this.items) {
      if (item.getId().equals(id)) {
        if (item.getStatus() == LendingStatus.AVAILABLE) {
          item.setStatus(LendingStatus.LENT);
          item.setBorrowerId(borrowerId);
          return true;
        }
      }
    }
    return false;
  }

  public boolean returnItem(String id) {
    for (T item : this.items) {
      if (item.getId().equals(id)) {
        if (item.getStatus() == LendingStatus.LENT) {
          item.setStatus(LendingStatus.AVAILABLE);
          item.setBorrowerId(null);
          return true;
        }
      }
    }
    return false;
  }

  public void showItems() {
    for (T item : this.items) {
      if (item.getStatus() == LendingStatus.LENT) {
        println("ID: " + item.getId() + ", タイトル: " + item.getTitle() + ", ステータス: " + item.getStatus() + " (利用者ID: " + item.getBorrowerId() + "), 最大貸出日数: " + item.getMaxLendingDays() + "日");
      } else {
        println("ID: " + item.getId() + ", タイトル: " + item.getTitle() + ", ステータス: " + item.getStatus());
      }
    }
  }
}
```

### 演習3の解答例
