# 授業資料 執筆スタイル・規約

『オブジェクト指向プログラミング』の授業資料を作成する際の執筆スタイルおよびルールを以下にまとめます。

## 1. 全体構成と見出し
* **大見出し (H1)**: 各回のタイトルに使用します（例: `# オブジェクト指向の基礎(1)：カプセル化`）。ページごとに1つだけ配置します。
* **中見出し (H2)**: 各トピックのセクションに使用します（例: `## スタックの実装`、`## カプセル化`）。
* **小見出し (H3)**: セクション内のトピック分割に使用します。過度にセクションを分けないよう、原則として `###` までを階層の上限とします。

## 2. コードブロックと実行結果の表記
`gitbook-plugin-codeblock-filename` を活用し、ファイル名や実行結果を明確に表示します。

* **ファイル名を指定するコード**: 
  ````markdown
  ```java:Stack.java
  class Stack { ... }
  ```
  ````
* **ファイル名なしのコード**: 
  ````markdown
  ```java
  void setup() { ... }
  ```
  ````
* **実行結果の提示**: 
  ````markdown
  ```console:実行結果
  20
  30
  ```
  ````
  ※原則として、コードの直後にその実行結果をセットで提示します。
* **実行可能な実例**: 文法要素の紹介の後には、単なる断片ではなく `void setup()` を含み、コピー＆ペーストでそのまま動作を確認できる実例を必ず記述してください。

## 3. 文章のトーン＆マナー
* **文末表現**: 「〜です」「〜ます」「〜しましょう」といった**丁寧語**（「です・ます」調）で統一し、学習者に歩み寄るやさしいトーンにします。
* **1行1文の徹底**: 通常の文章では、1行につき1文（句点「。」で改行）を徹底します。これにより、モバイル端末での読みやすさや差分の管理をしやすくします。
* **文字装飾の制限**: 太字（`**`）は新出の用語や核となる重要なキーワードの強調のみに使用します。文章の強調など、それ以外の目的では使用しないでください。
* **警告・注意の表記**: `> [!IMPORTANT]` などの GitHub スタイルの警告ブロックは使用できません。重要な注意点は、通常の文章の中で **【注意】** や太字の見出しを用いて記述してください。

## 4. 説明の流れ（段階的なステップアップ）
オブジェクト指向の概念やメリットを伝えるため、**「問題の発生 → 手順を踏んで改善」**というストーリー仕立てで構成します。

1. **素朴な実装と課題の提示**: まず基本的なコード（例: 変数ベタ書き）を見せ、バグが発生しやすいなどの「暗黙のルールに頼る危うさ」を指摘します。
2. **改善案の提示**: より良い方法（例: 関数化やクラス化）を提示し、コードを洗練させます。
3. **最終的な解決策の提示**: さらに残る問題点（例: 外部から破壊可能である点）を提示し、目当ての概念（カプセル化など）を用いて完全に解決した完成形を提示します。

## 5. 前提とするプログラミング環境
* 本資料のコード例は **Processing（Java）** をベースに記述します。
* メインの処理は `void setup()` 内に記述し、出力には `println()` を使用します。

## 6. 演習問題の記述
* **演習の問題文**: 「〜を述べなさい」「〜を作成しなさい」といった**命令調**で記述します。
* **回答例**: 丁寧語（です・ます）ではなく、**「である」調**で記述します。
